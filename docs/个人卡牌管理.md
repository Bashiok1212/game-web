# 个人卡牌管理（独立页）

- **不在主页展示**：`public/index.html` 未添加任何入口链接。
- **同一域名访问**：部署后地址为  
  `https://你的域名/ptcg.html`  
  例：`https://yaoling151.com/ptcg.html`
- **独立管理员**：与游戏账号无关，账号存 **MongoDB**（模型 `PtcgAdmin`）。页面 **注册** 或 **登录**。
  - `.env` 中 `PTCG_ADMIN_USER` / `PTCG_ADMIN_PASSWORD`：由启动脚本同步到 MongoDB，与「仅库中账号登录」一致。
  - 注册规则：若设置 `PTCG_REGISTER_SECRET`，注册时必须填写相同密钥；未设置时**仅允许第一个账号**注册。
  - 登录成功后令牌存 **sessionStorage**（关标签页即失效），约 7 天；JWT 内含 **`adminId`**，用于关联卡牌。
- **接口**：注册/登录/校验见 `API.md` 1.5；卡牌 CRUD 见 `API.md` 1.5 卡牌接口。
- **数据存储**：卡牌存 **MongoDB**（模型 `PtcgCard`，按 `admin` 隔离）。首次上线若本机仍有旧版 **localStorage**（`personalCards_v1`），页面会在列表为空时尝试 **导入到服务端** 后清除该键。仍建议定期 **导出 JSON** 备份。
- **隐私**：页面含 `noindex`，降低被搜索引擎收录的概率（勿在页面存敏感信息）。

## 文件

| 文件 | 说明 |
|------|------|
| `public/ptcg.html` | 页面结构 |
| `public/css/cards.css` | 样式（引用 `css/style.css` 变量） |
| `public/js/cards-app.js` | 增删改查、搜索、导入导出、入库/出库登记与明细、按配置渲染下拉/手输 |
| `public/ptcg-fields.html` | 固定字段的下拉选项编辑；版本按语言维护，每项含名称/年份/扩展编号 |
| `public/js/ptcg-fields-app.js` | 保存/读取 `field-dropdowns` |

## 功能概要

- 添加 / 编辑 / 删除卡牌，字段包括：**编号 `cardNo`（自动生成）**、名称、年份、语言、**版本名称**、**扩展编号 `versionCode`**、**版本编号 `versionNumber`**（每张卡在该扩展内的序号，如 `078/207`）、稀有度、购入价、是否评级卡（及评级公司/编号）、品相、备注、卡状态、图片（外链或本地上传 Base64）
- **数据模型**：每条记录对应**一张独立卡牌**，无「数量」；多张请添加多条记录。
- **入库 / 出库**：**新建卡牌**时会自动记一条入库（备注「添加卡牌」）；列表每行也可再登记「入」「出」，并可选同步更新卡状态；工具栏「出入库明细」查看全部登记记录（模型 `PtcgStockMovement`）。
- **编辑字段（独立页）**：`ptcg-fields.html` 为卡状态、稀有度、品相配置下拉；**语言与版本**按语言分栏，每种语言下可为每个版本填写**名称、对应年份、扩展编号、编号上限**（与「全局版本」兜底格式相同）。**版本行、语言块**均可用 ↑↓ **自定义排序**（保存后卡牌页下拉顺序与此一致；`language` 数组顺序与语言块一致以便下次加载）。录入页在配置了 `versionByLanguage` 时先选语言再选版本；从下拉选择版本时可自动带出**年份**与**扩展编号**到表单（仍可手改），下拉项中会显示编号上限。**保存并继续** / **Ctrl+Enter** 会清空名称、备注、图等并保留年份/语言/版本等便于连录。
- 搜索过滤（编号、名称、年份、语言、版本、稀有度、状态等）
- 导出 JSON、导入 JSON（合并导入；兼容旧版仅含 `set` 的导出）
