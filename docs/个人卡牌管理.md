# 个人卡牌管理（独立页）

- **不在主页展示**：`public/index.html` 未添加任何入口链接。
- **同一域名访问**：部署后地址为  
  `https://你的域名/ptcg.html`  
  例：`https://yaoling151.com/ptcg.html`
- **独立管理员**：与游戏账号无关，账号存 **MongoDB**（模型 `PtcgAdmin`）。页面 **注册** 或 **登录**。
  - 可选备用：`.env` 中 `PTCG_ADMIN_USER` / `PTCG_ADMIN_PASSWORD`（无 Mongo 账号时仍可用）。
  - 注册规则：若设置 `PTCG_REGISTER_SECRET`，注册时必须填写相同密钥；未设置时**仅允许第一个账号**注册。
  - 登录成功后令牌存 **sessionStorage**（关标签页即失效），约 7 天。
- **接口**：`POST /api/ptcg/register`、`POST /api/ptcg/login`、`GET /api/ptcg/verify`（见 `API.md` 1.5）。
- **数据存储**：卡牌数据仍仅存浏览器 **localStorage**（本机），清除站点数据会丢失，请定期 **导出 JSON** 备份。
- **隐私**：页面含 `noindex`，降低被搜索引擎收录的概率（勿在页面存敏感信息）。

## 文件

| 文件 | 说明 |
|------|------|
| `public/ptcg.html` | 页面结构 |
| `public/css/cards.css` | 样式（引用 `css/style.css` 变量） |
| `public/js/cards-app.js` | 增删改查、搜索、导入导出 |

## 功能概要

- 添加 / 编辑 / 删除卡牌（名称、系列、数量、品相、备注）
- 搜索过滤
- 导出 JSON、导入 JSON（合并导入）
