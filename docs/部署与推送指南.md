# 部署与推送指南

妖灵151 项目在云服务器上的部署、更新与本地推送说明。

---

## 一、仓库信息

| 项目 | 说明 |
|------|------|
| 仓库地址 | https://github.com/Bashiok1212/game-web |
| 默认分支 | main |
| 云服务器项目目录 | `/root/web` |

---

## 二、本地推送（手动）

### 1. 提交并推送到 main

```bash
cd h:\new-game          # 或你的项目根目录

# 查看修改
git status

# 添加要提交的文件
git add .
# 或只添加指定文件：git add server.js public/

# 提交
git commit -m "feat: 描述本次修改"

# 推送到 main（GitHub 默认分支）
git push origin main
```

### 2. 若本地分支是 master

```bash
git add .
git commit -m "feat: 描述本次修改"
git push origin master:main
```

### 3. 首次推送或设置上游

```bash
git push -u origin main
```

之后可直接使用 `git push`。

---

## 三、云服务器：拉取最新代码

### 方式一：使用部署脚本（推荐）

```bash
cd /root/web
bash deploy-pull.sh
```

脚本会自动完成：拉取代码 → 检查 .env → 安装依赖 → 重启 pm2。

### 方式二：手动拉取

```bash
cd /root/web
git fetch origin
git pull origin main
npm install --production
pm2 restart game-web
```

---

## 四、服务器重启后

### 若已配置 pm2 开机自启

服务器重启后，pm2 会自动拉起 game-web，一般无需操作。

### 若未配置或需手动启动

```bash
cd /root/web
pm2 start server.js --name game-web
pm2 save
pm2 startup   # 按提示执行输出的命令，完成开机自启
```

---

## 五、部署验证（排查 404）

若游戏丢弃道具返回 404，说明生产环境代码未更新。在**服务器**执行：

```bash
cd /root/web
bash scripts/verify-deploy.sh
```

脚本会检查 server.js、auth.js 是否包含 discard 路由，并测试本地 API。若有 ✗，执行：

```bash
bash deploy-pull.sh
```

然后浏览器访问 `https://yaoling151.com/api/health`，应返回 `{"ok":true,...}` 的 JSON。若仍是 HTML，检查：

1. **本地是否已推送**：`git status` 确认无未提交修改，`git push origin main` 已执行
2. **服务器拉取是否成功**：`git log -1` 查看当前 commit 是否与 GitHub 一致
3. **pm2 是否重启**：`pm2 restart game-web` 后 `pm2 logs game-web` 查看是否有报错

## 六、常用命令速查

| 操作 | 命令 |
|------|------|
| 本地推送 | `git add .` → `git commit -m "说明"` → `git push origin main` |
| 服务器拉取 | `cd /root/web` → `bash deploy-pull.sh` |
| 手动拉取 | `cd /root/web` → `git pull origin main` → `npm install --production` → `pm2 restart game-web` |
| 查看服务状态 | `pm2 list` |
| 查看日志 | `pm2 logs game-web` |
| 重启服务 | `pm2 restart game-web` |

## 七、首次部署（新服务器）

```bash
# 1. 克隆仓库
git clone https://github.com/Bashiok1212/game-web.git /root/web
cd /root/web

# 2. 安装依赖
npm install --production

# 3. 配置环境（自动生成 .env 和 JWT_SECRET）
node scripts/ensure-env.js

# 4. 确保 MongoDB 已启动
systemctl start mongod   # 或 docker start mongo

# 5. 启动服务
pm2 start server.js --name game-web
pm2 save
pm2 startup
```
